## 기획 완성도

- 사진·위치·일정 등 개인 일상 데이터를 기반으로 장기기억형 AI 회고 서비스를 제공하려는 문제 정의는 명확함. 다만 초기 사용자가 민감한 데이터 접근 권한을 허용할 만큼의 즉시 체감 가치가 중요하므로, 무료 장기기억 기능에서 어떤 경험을 제공하고 어떤 시점에 유료 기능으로 전환할지 구체화되면 좋겠음.
- 디지털 저널링에 대한 니즈와 확장성에 대해서 잘 정의하였음.
- 사진, 위치, 일정 등 모바일 기기 내 여러 서비스에 파편화되어 분산된 사용자의 생활 데이터를 자동으로 통합하여 하루의 타임라인을 구조화하는 아이디어가 우수함. 다만 사용자의 민감한 개인 데이터를 수집하기 위해 필수 권한을 획득하는 단계에서의 허들 극복용 UX 설계와 기기별 백그라운드 데이터 유실 방지 가동 시나리오 보완이 필요함.
- 사용자의 모바일 기기 데이터를 활용해 라이브 로그를 생성해 주는 유사한 과제, 서비스 시도는 많았으나 성공적인 사례는 많지 않음. 본 과제만의 차별성과 독창성을 고민해 보기 바람.
- 개인 일기 시장 자체는 크고 관련 앱들이 계속 출시되고 있어 타겟팅이 잘 되어 있다고 판단됨. 이 시장을 공략하기 위한 차별화 전략도 잘 제안되어 있음. 다만 설정한 전략이 과연 유효할 것인가에 대해서는 의구심이 들고 지속적으로 사용자 관점에서 고민해 주면 좋은 결과가 있을 것으로 판단됨.

## 아이디어의 우수성

- 단순 일기 작성이나 사진 정리를 넘어 사용자의 일상 데이터를 장기기억으로 축적하고, 이후 회고·검색·대화형 분석으로 연결하려는 점은 차별화 가능성이 있음. 다만 Apple Journal, Day One 등 기존 기록 서비스 대비 사용자가 별도 앱에 장기 데이터를 맡겨야 하는 이유와 신뢰 형성 방식이 더 명확해지면 좋겠음.
- 20대, 30대의 젊은 직장인보다는 50대 이상의 액티브 시니어 중심으로 피봇팅하면 어떨까라는 의견을 드림. 일기를 뛰어넘은 자서전 만들기 등의 신규 아이템도 가능해 보임.
- 사용자의 직접적인 입력 행위에 의존하여 지속성이 낮고 기록 피로도가 높았던 기존 저널링 앱들의 한계를 생성형 AI 기반의 자동 라이프 로깅 메커니즘으로 역발상 전환한 점이 우수함.
- 단순한 데이터 요약을 넘어 장기 축적된 데이터를 기반으로 기억 복원, 패턴 분석, 변화 인식 등 다각도 회고 기능을 제안하고 삶 전체 맥락 기반의 일상 AI 대화로 연결한 포지셔닝이 우수함.
- 자동으로 수집하여 활용할 수 있는 라이프 로깅 정보가 실제로 많지 않음. 사진이나 위치, 일정 정보만으로 어떤 결과물을 제공해 줄 수 있을지 추가 고민해 보기 바람.
- "수동 입력 저널링을 자동 수집으로 대체"라는 차별점은 명확하나, Apple Journal이 이미 사진·위치·운동 기반 추천을 제공하고 있으며 자동 기록 기능도 제공하고 있으나 지속적인 사용이 이루어지지 못하고 있어 지속적인 사용을 위한 고민이 필요함.

## 기술개발 가능성

- 사진·위치·일정 데이터를 연결해 하루 타임라인을 구성하고, 장기기억 기반 회고와 AI 대화를 제공하는 기술 구조는 서비스 목적과 부합함. 다만 자동으로 생성된 기억이 실제 사용자 맥락과 다를 수 있어, 사용자가 쉽게 수정·삭제·비공개 처리할 수 있는 UX와 온디바이스 처리 범위, 외부 모델 전송 기준이 보강되면 좋겠음.
- Linux Ubuntu 환경 내에서 Android 클라이언트 및 Spring Boot 백엔드, Kafka 메시징 시스템을 매칭하고 자체 에이전트 및 상용 LLM API를 복합 구성한 아키텍처 계획이 체계적으로 보임.
- 민감정보 보호를 위해 Edge SLM 및 On-device AI를 활용해 기기 내부에서 1차 가공 처리를 수행하도록 가드레일을 설계했으나, 안드로이드 버전별 백그라운드 구동에 따른 배터리 소모 제어의 실현 가능성 검증이 필수적으로 보임.
- 현재 제안된 내용으로는 단순 모바일 정보들을 추출해서 이를 통해 하루의 기록을 시각적으로 제공해 주는 것으로 보임. AI 기능을 어떻게 활용할 것인지, 쉽게 접근할 수 없는 정보를 본 과제에서 특별히 어떻게 접근할 것인지, 기존 정보들을 조합해 어떻게 새로운 정보를 도출해 낼 것인지 고민해 보기 바람.
- On-device SLM(Gemma·Phi)으로 사진·위치 1차 가공 후 서버 LLM으로 회고 생성하는 하이브리드 구조는 프라이버시·비용 측면에서 합리적임. 다만 모바일 On-device SLM에서 한국어 일상 이벤트 요약·감정 추출과 같은 부분의 성능이 제한적일 수 있어 최적화 이슈, 배터리 이슈가 있을 수 있으므로 고려가 필요함.

## 기대효과

- 장기기억 기능을 무료로 제공해 초기 사용 습관과 데이터 축적을 유도하고, 그 위에 고급 회고·생활 패턴 분석·AI 대화·과거 기억 검색 등을 유료화하는 전략은 합리적임. 다만 기록·회고 앱은 장기 리텐션 확보가 어려운 편이므로, 무료 사용자의 반복 방문률과 유료 기능 전환 포인트를 정량적으로 검증하면 좋겠음.
- 유저의 생활 기록이 아카이브로 누적될수록 분석 품질과 서비스 종속성이 동반 상승하여 강력한 장기 리텐션을 확보할 수 있으며, 3단계 요금제 등급 차등화 구독 모델 설계를 통해 지속 가능한 비즈니스로 성장할 수 있을 거라 기대됨.
- 개인의 기록을 하는 것은 매우 필요성이 높고 향후 일기뿐 아니라 업무 정리, 약속 정리 등으로 확장이 가능해 보임. 다만 일기 자체만으로 사용자의 리텐션을 얻기 쉽지 않아 보임.
