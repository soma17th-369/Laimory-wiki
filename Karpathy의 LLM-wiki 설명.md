# `llm-wiki-karpathy.md` 설명

## 이 파일은 무엇인가

`llm-wiki-karpathy.md`는 Andrej Karpathy가 제안한 LLM Wiki 패턴을 이 저장소에서 참고하기 쉽게 요약한 로컬 참조 문서다.

원문 gist를 그대로 복사한 파일은 아니고, 이 vault가 왜 이런 구조를 가져야 하는지 설명하는 짧은 운영 철학 문서에 가깝다. 즉 "LLM Wiki가 무엇인지"를 기억하기 위한 기준점이다.

## 핵심 메시지

이 파일의 핵심은 다음 한 문장으로 정리할 수 있다.

LLM은 질문할 때마다 원문을 새로 뒤지는 검색기가 아니라, 시간이 지날수록 더 좋아지는 마크다운 위키를 유지하는 관리자가 되어야 한다.

일반적인 RAG 방식은 질문이 들어올 때마다 원문 파일에서 관련 조각을 찾아 답을 만든다. 반면 LLM Wiki 방식은 새 자료가 들어올 때마다 LLM이 그 자료를 읽고, 기존 위키에 통합하고, 관련 페이지를 갱신한다. 그래서 지식이 채팅 기록에 흩어지지 않고 위키 안에 남는다.

## 문서가 설명하는 구조

`llm-wiki-karpathy.md`는 Karpathy의 LLM Wiki를 세 층으로 설명한다.

1. Raw sources

   사람이 제공한 원본 자료다. 기사, PDF, 웹 클립, GitHub README, 개인 노트 등이 여기에 들어간다. 이 자료는 원본 기록이므로 수정하지 않는 것이 원칙이다.

2. Wiki

   LLM이 유지하는 마크다운 지식 계층이다. 원문을 읽고 만든 소스 요약, 주제별 정리, 엔티티 페이지, 답변 페이지 등이 여기에 쌓인다.

3. Schema

   LLM에게 이 저장소를 어떻게 운영해야 하는지 알려주는 규칙 파일이다. 이 vault에서는 `AGENTS.md`가 schema 역할을 한다.

## 주요 작업 방식

문서는 LLM Wiki의 운영을 세 가지 작업으로 나눈다.

### Ingest

새 자료가 들어오면 LLM이 원문을 읽고, 요약하고, 관련 지식 페이지에 통합한다. 단순히 파일을 저장하는 것이 아니라 위키의 기존 구조에 연결하는 것이 중요하다.

### Query

질문을 받을 때는 원문을 매번 처음부터 뒤지기보다, 먼저 유지된 위키를 읽고 답한다. 좋은 답변은 다시 위키에 저장해 재사용 가능한 지식으로 만든다.

### Lint

위키가 커지면 주기적으로 점검한다. 모순, 오래된 주장, 고아 페이지, 빠진 링크, 중요한 공백을 찾아 관리한다.

## `index.md`와 `log.md`가 중요한 이유

이 파일은 `index.md`와 `log.md`를 작은 규모의 위키에서 핵심 내비게이션 장치로 본다.

- `index.md`: 위키 전체의 목차이자 지도
- `log.md`: 어떤 작업이 언제 이루어졌는지 남기는 시간순 기록

이 두 파일이 잘 유지되면 별도의 복잡한 검색 인프라 없이도 LLM이 위키를 꽤 잘 탐색할 수 있다.

## 이 vault에서의 의미

`llm-wiki-karpathy.md`는 직접 작업 규칙을 세세하게 정의하는 파일이라기보다, `AGENTS.md`가 왜 그런 규칙을 갖는지 설명하는 배경 문서다.

이 vault에서는 다음 역할을 한다.

- Karpathy LLM Wiki 패턴의 핵심을 기억하게 해준다.
- `raw/`, `wiki/`, `references/`, `AGENTS.md` 구조의 이유를 설명한다.
- RAG와 LLM Wiki 방식의 차이를 다시 확인하게 해준다.
- Obsidian을 LLM Wiki의 탐색/검토 인터페이스로 쓰는 관점을 제공한다.

## 언제 읽으면 좋은가

다음 상황에서 이 파일을 읽으면 좋다.

- 이 저장소가 왜 `raw/`와 `wiki/`를 나누는지 헷갈릴 때
- LLM Wiki 방식이 단순 문서 검색과 어떻게 다른지 다시 확인하고 싶을 때
- `AGENTS.md`의 운영 규칙을 수정하기 전에 상위 철학을 확인하고 싶을 때
- 새로운 사람이나 에이전트에게 이 vault의 기본 개념을 설명하고 싶을 때

## 주의할 점

이 파일은 원문 자료 자체가 아니라 참조/해설 문서다. 따라서 Karpathy gist의 정확한 표현을 인용하거나 원문 수준의 세부 맥락이 필요할 때는 gist 원문을 확인해야 한다.

또한 이 파일 자체를 외부 소스로 ingest하는 것보다, 이 파일은 `references/` 성격의 운영 참고 문서로 보는 것이 자연스럽다.

## 한 줄 요약

`llm-wiki-karpathy.md`는 이 vault가 Karpathy식 LLM Wiki로 작동해야 한다는 철학과 기본 구조를 설명하는 참조 문서다.
