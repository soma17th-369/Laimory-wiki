# Notion 369팀 Specification

## Summary

기능 명세 작성 가이드와 Laimory 주요 스펙 페이지 링크를 담은 Specification 허브.

## Key Claims

- 기능 명세는 카테고리, Depth_1~3, Priority, Apply, Status, MVP, Description 컬럼으로 관리한다.
- Priority는 매우높음/높음/보통/낮음/매우낮음으로 나뉜다.
- Status는 Backlog, Design, OnTrack, QA, Done, COMPLETE, ISSUE, HOLDING을 사용한다.
- MVP는 Open과 Update로 구분한다.
- 연결된 주요 문서는 `Epic - 시스템 초기 설계 및 구축`, `AI 하루 타임라인 기능 MVP 개발`이다.

## Caveats

- 이 페이지는 명세 작성 운영 규칙에 가깝고, 실제 기능별 상세 row는 별도 DB/페이지에 있을 수 있다.
